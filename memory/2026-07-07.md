# 2026-07-07 — Initial landing page

## Goal
Set up `digster.github.io` as a personal landing page showcasing my other
GitHub sites. Must be a plain HTML site with **no build step**.

## What was done
- Probed all 61 `digster` repos for live GitHub Pages deployments; found **9
  live sites** and curated their titles/descriptions from the live pages.
- Built a dependency-free static site served from the repo root:
  - `index.html` — hero (avatar + handle + tagline + GitHub link), search bar,
    card grid container, footer. Inline no-flash theme boot script in `<head>`.
  - `assets/style.css` — playful & colorful theme; per-card accent hue driven by
    an inline `--h` custom property; animated gradient hero title + floating
    blobs; light/dark tokens; respects `prefers-reduced-motion`.
  - `assets/app.js` — fetches `data/sites.json`, renders cards, live text search
    + click-to-filter tag chips, persisted light/dark toggle, load-error state.
  - `data/sites.json` — the 9 sites (single source of truth for the gallery).
- Supporting files: `.nojekyll`, `README.md`, `ARCHITECTURE.md`, `.gitignore`,
  MIT `LICENSE` (holder: digster), empty `CLAUDE.md`, `PROMPT.md`, this memory.

## Design choices (confirmed with user)
- Style: **playful & colorful**; Scope: **9 live sites only**;
  Header: **handle + builder tagline**.

## Key decisions / notes
- Content is data-driven via `data/sites.json` so adding a site = editing one
  file (no code, no build). Rendered client-side with `fetch()`.
- Card colors are derived from the first tag (`TAG_HUES` map in `app.js`) with a
  hash-based nudge so same-tag cards stay distinct.
- Chose a curated JSON over live GitHub API calls (API can't reliably detect
  Pages from the client + rate limits).
- Because of `fetch()`, local preview needs a static server
  (`python3 -m http.server`), not `file://`.
